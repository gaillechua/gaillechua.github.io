    More like a disarray and clashes
    October 3, 2014

    While researching on the topic of arrays and hashes, I could not help but start to think ahead about what analogy to use because I know that would be beneficial to both me and you (the reader) the most. I was jumping ahead of myself, though, realizing that most resources that describe the difference are indeed authored by programmers themselves, therefore have very candid and direct explanations. So, I'd like to first explain them as straightforwardly as I can.

    WHAT THEY HAVE IN COMMON

    • ARRAYS & HASHES
    • - They store collections of objects that are accessible with a key
    • - They hold different object types: integers, strings, floats, booleans, etc.
    • - They are indexed using the same square brack notation '[ ]'
    • - They both store data

    THE DIFFERENCES

    • ARRAYS
    • - More efficient to access array elements
    • - Ordered, integer-indexed collections of any object
    • - Enumerate their values according to the integer they are indexed in. The integer '0' is the first index, therefore '1' is the second.
    • - Each object in an array has a unique key assigned to it. We can access any object in the array using this unique key.
    • HASHES
    • - Provide more flexibility
    • - A collection of key-value pairs, but indexing is done via arbitrary keys of any object type, not an integer index
    • - Enumerate their values in the order that the corresponding keys were inserted
    • - A hash literal uses braces rather than square brackets. The literal must supply two objects for every entry: one for the key, the other for the value
    • - Hashes point an object to another object for "meaning."

    THE ANALOGY

    Let's use analogies of cars again, because it's always the first thing that comes to mind :)

    An array is a unique list of parts that allows you to access a unique part of the car. And let's say they have to be done sequentially before you get jetting: (0) ignition key, (1) seatbelt, (2) clutch/gear shifter, (3) brake pedal, (4) gas pedal.

    A hash is a collection of key/value pairs: (person) Gaille has a (car model) Subaru Impreza, (person) Renate has a (car model) Volkswagen GTI, (person) Eric has a (car model) Scion TC.

    Eeps. I hope that makes sense?

    So, let's go one step further to solidify our understanding. I'll format it so that it's readable from a programmer's perspective (and gives me more practice with Ruby syntax), but is still understandable from a non-technical person's point of view (which is admittedly me still).

    • ARRAY
    • carparts = Array.new
    • carparts = ([0]ignition key, [1] seatbelt, [2] clutch/bear shifter, [3] brake pedal, [4] gas pedal

    • Therefore, carparts[3] = "brake pedal" (which is the 4th step in our array; remember 0 is the starting enumerator!)

      • HASH
    • carowners = Hash.new
    • carowners = {'Gaille' => 'Subaru Impreza', 'Renate' => 'Volkswagen GTI', 'Eric' => 'Scion TC'}

    • Therefore, carowners['Renate'] => 'Volkswagen GTI'
