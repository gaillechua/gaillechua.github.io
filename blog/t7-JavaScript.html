    Ruby vs. JavaScript: What's So Different?
    Preventing syntax confuzzledness
    November 1, 2014

    The beauty of Ruby and JavaScript is that there are a myriad of ways to define, execute, and write them to reach the same goal. The flexibility is mostly great. We have the freedom to determine what way is the most suitable and preferential to use for our work. But it is also that awesome wizardy and syntax similarity that can cause confusion. Because the differences are less noticeable between languages, those subtleties can inevitably lead to numerous, frustrating, face-palming errors in code.

    I think that confusion becomes inadvertently compounded for DBC students. Because our program is accelerated, it's easy to fall into a pit of confusing similar-looking languages because it all looks new to most of us. I may confuse how to declare a function vs declaring a method in Ruby, for example. Do I end a function using "end" or do I use a semi-colon? Moreover, what does "break" do in Ruby and JavaScript and can I apply it logically in the same bits of code? The list goes on, but the point is it's good to call this out early in order to have a firm understanding of writing code without leading to early errors.

    My GPS guide for this week mentioned that a lot of the syntax and code-writing will become muscle memory at some point. I think that a quick way to get to that point is to recognize the differences and write them out using examples from both Ruby and JavaScript. I will add more as I have time.

    So, while I think that some similarities are helpful in understanding concepts and logic of different coding languages in general, I also do think that similarities can cause confusion and it's better to nip it in the bud by learning the subtle differences now so that it becomes muscle memory sooner rather than later and so that the syntaxes don't get jumbled later down the road.
